③ AWS 엔드포인트 통신 확인 (코드 셀 3)
자격증명이 없으니 인증 에러가 나는 게 정상입니다. 에러 종류로 네트워크가 닿는지를 판별해요.

import urllib.request

targets = {
    'STS':            'https://sts.amazonaws.com/',
    'Bedrock (us-east-1)':  'https://bedrock-runtime.us-east-1.amazonaws.com/',
    'PyPI':           'https://pypi.org/',
    'GitHub Raw':     'https://raw.githubusercontent.com/',
}

for name, url in targets.items():
    try:
        urllib.request.urlopen(urllib.request.Request(url, method='HEAD'), timeout=5)
        print(f'OK   {name}')
    except urllib.error.HTTPError as e:
        # 4xx 응답이 와도 통신이 닿은 것 → 정상
        print(f'OK   {name} (HTTP {e.code})')
    except Exception as e:
        print(f'FAIL {name} -> {type(e).__name__}')
✅ 성공 기준 4개 항목 모두 OK로 출력되면 성공입니다. (응답 코드가 400/403으로 보여도 통신이 닿은 거라 OK)
💡 막혔다면? ① 특정 항목에 FAIL ... TimeoutError / URLError가 나오면, 해당 도메인이 차단됐을 가능성이 큽니다. (Colab은 구글 서버에서 동작하므로 보통 잘 됩니다)
② Colab 자체에 접속이 안 된다면 STEP 1로 돌아가 주세요.
③ !pip install이 실패하면 에러 메시지 첫 줄을 챗봇에 붙여넣어 주세요.
🗓 교육 당일에는? 교육 시작 시 안내드리는 임시 자격증명(또는 API 키)을 코드 셀에 넣으면 위에서 통신을 확인한 엔드포인트로 실제 모델 호출이 가능합니다. 자격증명은 보안상 사전 배포하지 않으니, 사전에는 "통신이 닿는지"까지만 확인하시면 충분합니다.

?자주 묻는 질문 (FAQ)

막혔을 때 가장 자주 발생하는 케이스들입니다. 챗봇에도 같은 내용이 학습되어 있어요.

Q1. Colab 페이지가 아예 안 열려요
사내 망에서 colab.research.google.com이 차단되어 있을 가능성이 가장 큽니다. 사내 IT 담당자에게 해당 도메인 방화벽 허용을 요청해 주세요. 임시로는 모바일 핫스팟이나 개인 와이파이로 우회해서 테스트해볼 수 있습니다.
Q2. AWS 로그인 시 "잘못된 자격증명" 에러가 나요
계정 ID(숫자 12자리), 사용자 이름, 비밀번호를 한 글자씩 다시 확인해 주세요. 특히 "루트 사용자"가 아닌 "IAM 사용자"로 로그인해야 합니다. 화면 상단의 토글을 확인해 주세요.
Q3. Bedrock 콘솔에 들어가면 "Service not available in this region"이 떠요
Bedrock은 일부 리전에서만 제공됩니다. 우측 상단 리전 선택에서 "US East (N. Virginia) us-east-1" 또는 "US West (Oregon) us-west-2"로 변경해 보세요. 교육 담당자가 안내한 리전을 사용하시면 됩니다.
Q4. AccessDeniedException 에러가 나요
권한이 부족해서 발생합니다. 확인할 점:
① Bedrock Console → Model access에서 해당 모델이 "Access granted" 상태인지
② IAM 사용자/역할에 bedrock:InvokeModel, bedrock:ListFoundationModels 권한이 있는지
③ 호출하는 리전과 모델 액세스 승인 리전이 같은지
해결되지 않으면 챗봇에 에러 메시지 전문을 붙여넣어 주세요.
Q5. UnrecognizedClientException 또는 InvalidClientTokenId 에러가 나요
사전 테스트에서는 자격증명을 입력하지 않으므로 이 에러는 보통 교육 당일에 마주치게 됩니다. AWS 자격증명(Access Key / Secret Key)에 오타가 있거나 임시 자격증명의 Session Token이 누락된 경우입니다. 현장에서 안내받은 키 3개(Access Key / Secret Key / Session Token)를 모두 복사해서 다시 넣어주세요. 키 값 앞뒤에 공백이나 줄바꿈이 섞이지 않도록 주의합니다.
Q6. pip install이 사내에서는 안 돼요
Colab은 구글 서버에서 실행되므로, 사내 망과 상관없이 보통 PyPI 접속이 잘 됩니다. 로컬 PC에서 pip 명령어를 직접 실행하는 게 아니라, Colab 안에서 !pip install 을 실행하는 거예요. Colab 코드 셀에 !pip install boto3처럼 느낌표(!)를 붙여서 실행해 보세요.
Q7. list_foundation_models 호출은 되는데, invoke_model은 실패해요
모델 목록 조회 권한은 있지만, 실제 호출 권한이 없거나 모델 액세스가 승인되지 않은 상태입니다. Bedrock Console → Model access에서 호출하려는 모델 ID가 "Access granted"인지 확인해 주세요. 모델 ID는 정확히 일치해야 합니다 (예: anthropic.claude-3-haiku-20240307-v1:0).
Q8. 사전 테스트에 자격증명이나 API 키가 필요한가요?
아니요, 필요하지 않습니다. 사전 테스트는 "환경 접근성" 확인 단계라서, AWS Console 로그인은 본인 회사 계정으로 진행하시고 Colab에서는 자격증명 없이 통신만 확인합니다. 실습용 임시 자격증명(또는 API 키)은 보안상 사전에 배포하지 않고 교육 당일 현장에서 별도로 안내드립니다.
Q9. 사전 테스트는 언제까지 해야 하나요?
