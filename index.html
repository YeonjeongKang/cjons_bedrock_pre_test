1Google Colab 접속 확인

교육은 PC에 Python을 설치하지 않고, 브라우저에서 바로 코드를 실행하는 Google Colab으로 진행됩니다.

📔 새 노트북 만들고 코드 한 줄 실행해보기

⏱ 2분

Colab에 접속해서 새 노트북을 만들고, 간단한 코드가 실행되는지 확인합니다.

  1. 아래 버튼으로 Colab을 새 탭에서 열어주세요. (Google 계정 로그인 필요)
  2. 상단 메뉴 파일 → 새 노트를 클릭해서 빈 노트북을 생성합니다.
  3. 회색 코드 셀에 아래 코드를 입력하고, Shift + Enter로 실행해주세요.

✅ 성공 기준 코드 셀 바로 아래에 ok가 출력되면 성공입니다.
🚀 Colab 열기 Google 로그인
💡 막혔다면? ① 사내 망에서 Colab(colab.research.google.com)이 차단됐을 수 있어요. → 오른쪽 아래 챗봇에 "Colab 접속 안 됨"이라고 물어보세요.
② 회사 계정으로 Google 로그인이 안 된다면, 개인 Gmail로 로그인 후 새 탭에서 Colab을 다시 열어보세요.

2AWS Console 로그인 확인

교육용으로 발급된 AWS 계정에 로그인할 수 있는지 확인합니다. 본인 개인 AWS 계정이 아닌, 교육 담당자가 안내한 계정을 사용해 주세요.

🔐 교육용 AWS 계정으로 로그인해보기

⏱ 3분

AWS Console에 로그인하고, 교육에서 사용할 리전(Region)으로 전환할 수 있는지 확인합니다.

  1. 아래 버튼으로 AWS Console을 열어주세요.
  2. 교육 담당자가 안내한 계정 ID, 사용자 이름, 비밀번호를 입력합니다. (SSO 안내가 있다면 SSO 방식으로)
  3. MFA(2단계 인증) 코드를 요구하면 사내 가이드에 따라 입력해주세요.
  4. 로그인 성공 후, 우측 상단의 리전 이름(예: 서울, N. Virginia)을 클릭해서 교육 담당자가 안내한 리전으로 변경합니다.
📍 리전이 뭔가요? AWS 서비스가 운영되는 지역입니다. Bedrock은 리전마다 사용 가능한 AI 모델이 다르기 때문에, 교육에서 정한 리전을 꼭 맞춰주셔야 해요. 보통 "us-east-1 (버지니아 북부)" 또는 "us-west-2 (오리건)"을 사용합니다.
✅ 성공 기준 로그인이 되고, 우측 상단에서 리전을 변경할 수 있으면 성공입니다.
🚀 AWS Console 열기
💡 막혔다면? ① 로그인이 안 되면 계정 정보를 한 글자씩 확인해 보세요. (특히 계정 ID는 12자리 숫자입니다)
② MFA가 안 되면 사내 IT 담당자에게 MFA 디바이스 등록 상태를 확인해 주세요.
③ console.aws.amazon.com 자체가 차단됐다면, IT 담당자에게 "방화벽 화이트리스트" 요청이 필요합니다.

3Bedrock 모델 액세스 확인

Amazon Bedrock 콘솔에 들어가서, 교육에서 사용할 AI 모델들이 "사용 승인(Available)" 상태인지 확인합니다.

🤖 Claude, Titan 모델이 사용 가능한지 확인

⏱ 3분

Bedrock에서는 AI 모델을 사용하기 전에 "Model access(모델 액세스)" 승인을 받아야 합니다. 미리 승인되어 있는지 확인합니다.

  1. AWS Console에 로그인된 상태에서, 아래 버튼으로 Bedrock 콘솔을 열어주세요.
  2. 왼쪽 사이드바에서 "Model access"를 클릭합니다. (좀 아래쪽에 있어요)
  3. 모델 목록에서 다음 두 가지 항목이 "Access granted" 또는 "Available"로 표시되는지 확인합니다.
    • Anthropic Claude 계열 (Claude 3 Sonnet, Claude 3 Haiku 등 어떤 것이든)
    • Amazon Titan Embeddings (Titan Text Embeddings V2 등)
🎁 추가 확인: Playground에서 모델 호출해보기 Bedrock 콘솔 좌측 메뉴 "Playgrounds → Chat / Text"로 들어가서, Claude 모델을 선택하고 "안녕"이라고 입력해보세요. 응답이 오면 모델 액세스가 잘 된 상태입니다.
✅ 성공 기준 Claude와 Titan Embeddings 모델이 모두 "Access granted" 상태이면 성공입니다.
🚀 Bedrock 콘솔 열기
💡 막혔다면? ① 모델 상태가 "Request access"로 떠있다면, 교육 담당자에게 "Model access 승인 요청 부탁드린다"고 알려주세요. 승인까지 몇 시간 ~ 1일 걸릴 수 있어요.
② Bedrock 페이지 자체가 안 열린다면, 오른쪽 리전을 교육 지정 리전으로 변경했는지 확인하세요. 한국 리전(ap-northeast-2)에서는 Bedrock이 제공되지 않을 수 있어요.

4Colab에서 Bedrock API 호출 테스트

가장 중요한 단계입니다. Colab 노트북에서 Bedrock에 코드로 접속되는지 확인합니다.

🧪 패키지 설치 → 자격증명 → API 호출까지 한 번에

⏱ 5분

아래 코드를 차례대로 Colab의 코드 셀에 복사해서 실행해 주세요. 교육 담당자가 안내한 임시 AWS 자격증명(Access Key, Secret Key, Session Token)을 사용합니다.

① 필요한 패키지 설치 (코드 셀 1)

!pip install -q boto3 botocore

② AWS 자격증명 설정 (코드 셀 2) — 따옴표 안에 본인 키 입력

import os
os.environ['AWS_ACCESS_KEY_ID'] = '교육담당자가-알려준-Access-Key'
os.environ['AWS_SECRET_ACCESS_KEY'] = '교육담당자가-알려준-Secret-Key'
os.environ['AWS_SESSION_TOKEN'] = '교육담당자가-알려준-Session-Token'
os.environ['AWS_DEFAULT_REGION'] = 'us-east-1'  # 교육 지정 리전으로 변경

③ Bedrock 모델 목록 조회 (코드 셀 3) — 가장 쉬운 확인 방법

import boto3
client = boto3.client('bedrock')
models = client.list_foundation_models()
print(f"사용 가능한 모델 개수: {len(models['modelSummaries'])}")
print("첫 번째 모델:", models['modelSummaries'][0]['modelId'])

④ Claude에게 진짜로 질문 던져보기 (코드 셀 4)

import json
runtime = boto3.client('bedrock-runtime')

response = runtime.invoke_model(
    modelId='anthropic.claude-3-haiku-20240307-v1:0',
    body=json.dumps({
        "anthropic_version": "bedrock-2023-05-31",
        "max_tokens": 100,
        "messages": [{"role": "user", "content": "안녕! 한 문장으로 자기소개 해줘"}]
    })
)
result = json.loads(response['body'].read())
print(result['content'][0]['text'])
✅ 성공 기준 ③에서 모델 개수가 출력되고, ④에서 Claude가 자기소개 문장으로 응답하면 성공입니다.
💡 막혔다면? 에러 메시지의 첫 줄을 그대로 복사해서 챗봇에 붙여넣어 보세요. 자주 나오는 에러는 아래 FAQ에도 정리되어 있어요. 대표 케이스: AccessDeniedException, UnrecognizedClientException, ValidationException, EndpointConnectionError

?자주 묻는 질문 (FAQ)

막혔을 때 가장 자주 발생하는 케이스들입니다. 챗봇에도 같은 내용이 학습되어 있어요.

Q1. Colab 페이지가 아예 안 열려요
사내 망에서 colab.research.google.com이 차단되어 있을 가능성이 가장 큽니다. 사내 IT 담당자에게 해당 도메인 방화벽 허용을 요청해 주세요. 임시로는 모바일 핫스팟이나 개인 와이파이로 우회해서 테스트해볼 수 있습니다.
Q2. AWS 로그인 시 "잘못된 자격증명" 에러가 나요
계정 ID(숫자 12자리), 사용자 이름, 비밀번호를 한 글자씩 다시 확인해 주세요. 특히 "루트 사용자"가 아닌 "IAM 사용자"로 로그인해야 합니다. 화면 상단의 토글을 확인해 주세요.
Q3. Bedrock 콘솔에 들어가면 "Service not available in this region"이 떠요
Bedrock은 일부 리전에서만 제공됩니다. 우측 상단 리전 선택에서 "US East (N. Virginia) us-east-1" 또는 "US West (Oregon) us-west-2"로 변경해 보세요. 교육 담당자가 안내한 리전을 사용하시면 됩니다.
Q4. AccessDeniedException 에러가 나요
권한이 부족해서 발생합니다. 확인할 점:
① Bedrock Console → Model access에서 해당 모델이 "Access granted" 상태인지
② IAM 사용자/역할에 bedrock:InvokeModel, bedrock:ListFoundationModels 권한이 있는지
③ 호출하는 리전과 모델 액세스 승인 리전이 같은지
해결되지 않으면 챗봇에 에러 메시지 전문을 붙여넣어 주세요.
Q5. UnrecognizedClientException 또는 InvalidClientTokenId 에러가 나요
AWS 자격증명(Access Key, Secret Key)에 오타가 있거나, 임시 자격증명의 Session Token이 누락된 경우입니다. 교육 담당자가 전달한 키 3개(Access Key / Secret Key / Session Token)를 모두 복사해서 다시 넣어주세요. 또한 임시 자격증명은 만료 시간이 있어서, 시간이 지나면 새로 받아야 합니다.
Q6. pip install이 사내에서는 안 돼요
Colab은 구글 서버에서 실행되므로, 사내 망과 상관없이 보통 PyPI 접속이 잘 됩니다. 로컬 PC에서 pip 명령어를 직접 실행하는 게 아니라, Colab 안에서 !pip install 을 실행하는 거예요. Colab 코드 셀에 !pip install boto3처럼 느낌표(!)를 붙여서 실행해 보세요.
Q7. list_foundation_models 호출은 되는데, invoke_model은 실패해요
모델 목록 조회 권한은 있지만, 실제 호출 권한이 없거나 모델 액세스가 승인되지 않은 상태입니다. Bedrock Console → Model access에서 호출하려는 모델 ID가 "Access granted"인지 확인해 주세요. 모델 ID는 정확히 일치해야 합니다 (예: anthropic.claude-3-haiku-20240307-v1:0).
Q8. 임시 자격증명이 "만료됐다"고 나와요
교육용 임시 자격증명은 보통 몇 시간~하루 정도만 유효합니다. 만료되었다면 교육 담당자에게 새 자격증명을 요청해 주세요. 사전 테스트 후 시간이 지나서 교육 당일 다시 발급받아야 할 수 있습니다.
Q9. 사전 테스트는 언제까지 해야 하나요?
가능하면 교육 2~3 영업일 전까지 완료해 주세요. Bedrock Model access 승인이 즉시 안 될 수 있고, 사내 방화벽 허용 요청도 시간이 걸리기 때문입니다. 교육 전날까지 막힌 부분이 있다면 교육 담당자에게 미리 알려주세요.
Q10. 막혔는데 회사 내 누구한테 도움 요청해야 하나요?
① Colab/PyPI/GitHub 접속 차단 → 사내 IT/네트워크 담당자에게 방화벽 허용 요청
② AWS 계정 로그인/MFA 이슈 → 교육 담당자
③ Bedrock 모델 권한 이슈 → 교육 담당자
④ 코드 에러/사용법 → 우측 하단 챗봇에 먼저 물어보고, 안 풀리면 교육 담당자
